# Publish Risk Checklist

- [ ] Check whether the repository URL points to the intended package index.
- [ ] Confirm the workflow will not publish to the wrong destination.
- [ ] Review whether credentials are environment-specific or reusable.
- [ ] Confirm artifact generation happens before publish.
- [ ] Check whether package contents match the intended release scope.
- [ ] Record rollback, disable, or pause steps if the publish misfires.
- [ ] Note user impact if a bad release escapes.
- [ ] Flag any missing verification evidence as a release risk.
