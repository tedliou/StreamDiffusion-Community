# Release Checklist

- [ ] Confirm the intended version and tag format.
- [ ] Confirm the release workflow that will run for the tag.
- [ ] Review whether package metadata and release notes are up to date.
- [ ] Confirm build inputs and packaging commands are still valid.
- [ ] Check whether artifact names and expected outputs are known.
- [ ] Confirm the publish target is correct.
- [ ] Confirm the required secrets exist and are scoped correctly.
- [ ] List the verification steps after the release run.
- [ ] Note whether this is a dry run, TestPyPI publish, or production publish.
- [ ] Record any blocking risks before release.
